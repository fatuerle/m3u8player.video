How to Fix CORS Errors When Playing M3U8 Streams

You found the perfect M3U8 link, you pasted it into a web player, clicked play, and... nothing happened. If you press F12 to open your browser's developer console, you are likely staring at a wall of red text that says: "Access to fetch at [...] has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present on the requested resource."

Don't panic. This is the single most common issue when dealing with web-based HLS video streaming. In this guide, we will explain exactly what CORS is and provide step-by-step solutions for both regular users and server developers.

1. What is a CORS Error?

CORS stands for Cross-Origin Resource Sharing. It is a fundamental security mechanism built into all modern web browsers (like Chrome, Firefox, and Safari).

By default, browsers restrict web pages from making requests to a different domain than the one that served the web page. This is called the Same-Origin Policy. It exists to prevent malicious websites from silently stealing your data from other tabs you have open (like your bank account).

2. Why Does it Happen with M3U8 Playlists?

When you use a tool like M3U8Player.video (Domain A) to play a video stream hosted on an external CDN or IPTV server (Domain B), our player uses JavaScript (specifically `hls.js`) to request the .m3u8 playlist and the .ts video segments from Domain B.

Because Domain A and Domain B are different, your browser steps in. It asks Domain B: "Hey, is it okay if M3U8Player.video accesses this video file?"

If the server hosting the video (Domain B) does not explicitly reply with an HTTP header saying Yes (Access-Control-Allow-Origin: *), your browser blocks the video from loading to protect you. That block is the CORS error.

3. The Quick Fix: For Regular Users (Viewers)

If you are just trying to watch a stream and do not have access to the server hosting the video, you cannot change the server's HTTP headers. However, you can tell your browser to temporarily ignore the CORS security rules.

Use a CORS Bypass Browser Extension

The easiest workaround is installing a trusted browser extension that forces the Access-Control-Allow-Origin header onto every request.

  1. Open the Chrome Web Store and search for Allow CORS: Access-Control-Allow-Origin.
  2. Click Add to Chrome.
  3. Once installed, click the extension icon in your toolbar and toggle it ON (the icon usually turns colored instead of gray).
  4. Refresh the M3U8 Player page and try playing your link again. The video should now load instantly.

Security Note: Remember to turn the extension OFF when you are done watching your stream, as leaving it on permanently can expose your browser to security risks on other websites.

4. The Permanent Fix: For Server Admins & Developers

If you own the server or CDN hosting the M3U8 files and video segments, it is your responsibility to configure it to accept cross-origin requests so your users don't have to install sketchy browser extensions.

You need to add specific headers to your server configuration. The most important one is Access-Control-Allow-Origin. Depending on your web server software, here is how to apply it:

Fixing CORS in Nginx

Open your Nginx configuration file (usually located at /etc/nginx/nginx.conf or within your sites-available directory) and add the following headers inside the location block that serves your video files:

# Add inside your server block
location /hls/ {
    add_header 'Access-Control-Allow-Origin' '*' always;
    add_header 'Access-Control-Expose-Headers' 'Content-Length,Content-Range';
    add_header 'Access-Control-Allow-Headers' 'Range';

    # Handle preflight OPTIONS requests
    if ($request_method = 'OPTIONS') {
        add_header 'Access-Control-Allow-Origin' '*';
        add_header 'Access-Control-Allow-Headers' 'Range';
        add_header 'Access-Control-Max-Age' 1728000;
        add_header 'Content-Type' 'text/plain charset=UTF-8';
        add_header 'Content-Length' 0;
        return 204;
    }
}

Fixing CORS in Apache

If you are using Apache, you can add these headers via your .htaccess file or directly in your virtual host configuration. Ensure the mod_headers module is enabled.

<IfModule mod_headers.c>
    Header set Access-Control-Allow-Origin "*"
    Header set Access-Control-Allow-Headers "Range"
    Header set Access-Control-Expose-Headers "Content-Length,Content-Range"
</IfModule>

Fixing CORS in AWS S3 / CloudFront

If your HLS files are stored in an Amazon S3 bucket, go to the Permissions tab of your bucket, scroll down to Cross-origin resource sharing (CORS), and paste this JSON configuration:

[
    {
        "AllowedHeaders": ["*"],
        "AllowedMethods": ["GET", "HEAD"],
        "AllowedOrigins": ["*"],
        "ExposeHeaders": ["Access-Control-Allow-Origin"]
    }
]

Conclusion

CORS policies are a vital part of web security, but they can be a massive headache for video streaming. Whether you temporarily bypass the block using a browser extension as a viewer, or permanently fix your server headers as a developer, you should now be able to play your HLS streams flawlessly.
